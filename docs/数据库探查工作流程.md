# 数据库探查工作流程

> 适用于对接厂商数据库、表结构不明确、无注释的情况

## 1. 初始探查 — 摸清库的基本情况

### 1.1 查看所有表（含估算行数）

```sql
SELECT schemaname, tablename, n_live_tup AS 估算行数
FROM pg_stat_user_tables
ORDER BY schemaname, tablename;
```

### 1.2 查看所有 schema

```sql
SELECT schema_name
FROM information_schema.schemata
WHERE schema_name NOT IN ('pg_catalog', 'information_schema', 'pg_toast')
ORDER BY schema_name;
```

### 1.3 查看库中所有表（含注释，如果有的话）

```sql
SELECT
    schemaname AS 模式,
    tablename AS 表名,
    tableowner AS 所有者,
    n_live_tup AS 估算行数
FROM pg_stat_user_tables
ORDER BY schemaname, tablename;
```

## 2. 单表结构探查

### 2.1 查看字段名、类型、是否可空

```sql
SELECT
    ordinal_position AS 序号,
    column_name AS 字段名,
    data_type AS 数据类型,
    character_maximum_length AS 最大长度,
    is_nullable AS 可否为空,
    column_default AS 默认值
FROM information_schema.columns
WHERE table_schema = 'public'    -- 替换为实际 schema
  AND table_name = '你的表名'     -- 替换为实际表名
ORDER BY ordinal_position;
```

### 2.2 查看字段注释（如果厂商有写）

```sql
SELECT
    a.attname AS 字段名,
    pg_catalog.col_description(a.attrelid, a.attnum) AS 注释,
    pg_catalog.format_type(a.atttypid, a.atttypmod) AS 类型
FROM pg_catalog.pg_attribute a
WHERE a.attrelid = (
    SELECT c.oid
    FROM pg_catalog.pg_class c
    JOIN pg_catalog.pg_namespace n ON n.oid = c.relnamespace
    WHERE n.nspname = 'public'      -- 替换为实际 schema
      AND c.relname = '你的表名'      -- 替换为实际表名
)
  AND a.attnum > 0
  AND NOT a.attisdropped
ORDER BY a.attnum;
```

### 2.3 查看主键

```sql
SELECT
    kcu.column_name AS 主键字段
FROM information_schema.table_constraints tc
JOIN information_schema.key_column_usage kcu
    ON tc.constraint_name = kcu.constraint_name
WHERE tc.table_schema = 'public'
  AND tc.table_name = '你的表名'
  AND tc.constraint_type = 'PRIMARY KEY';
```

## 3. 表间关系探查

### 3.1 查看所有外键关系

```sql
SELECT
    tc.table_schema AS 源模式,
    tc.table_name AS 源表,
    kcu.column_name AS 源字段,
    ccu.table_schema AS 目标模式,
    ccu.table_name AS 目标表,
    ccu.column_name AS 目标字段
FROM information_schema.table_constraints tc
JOIN information_schema.key_column_usage kcu
    ON tc.constraint_name = kcu.constraint_name
JOIN information_schema.constraint_column_usage ccu
    ON tc.constraint_name = ccu.constraint_name
WHERE tc.constraint_type = 'FOREIGN KEY'
ORDER BY tc.table_schema, tc.table_name;
```

### 3.2 通过字段名推测关联关系

如果厂商没建外键，可以通过命名规律找关联键：

```
常见关联键命名规律：
  - 表A.id  ↔  表B.表A_id
  - patient_id  →  patient_master_info
  - inpat_id     →  case_base / inpat_record
  - case_id      →  case_operation / case_diagnose
  - dept_code    →  department
  - employee_id  →  employee
```

## 4. 字段含义推断 — 采样数据

### 4.1 采样前几行看数据

```sql
SELECT * FROM public.你的表名 LIMIT 20;
```

### 4.2 看某个字段的取值分布（离散值）

```sql
SELECT
    你的字段名,
    COUNT(*) AS 出现次数
FROM public.你的表名
GROUP BY 你的字段名
ORDER BY COUNT(*) DESC
LIMIT 30;
```

### 4.3 看某个字段的取值范围（连续值/日期）

```sql
SELECT
    MIN(你的字段名) AS 最小值,
    MAX(你的字段名) AS 最大值,
    COUNT(*) AS 非空数,
    COUNT(*) FILTER (WHERE 你的字段名 IS NULL) AS 空值数
FROM public.你的表名;
```

### 4.4 看日期/时间字段的区间

```sql
SELECT
    MIN(你的时间字段) AS 最早时间,
    MAX(你的时间字段) AS 最晚时间,
    COUNT(DISTINCT 你的时间字段::date) AS 涵盖天数
FROM public.你的表名;
```

```markdown

|------|--------|---------|---------|
| case_base | public | 50万 | 病案首页主表 |
| ... | ... | ... | ... |

## 各表字段详情

### public.case_base
| 字段名 | 类型 | 推测含义 | 依据 |
|--------|------|---------|------|
| case_id | bigint | 主键ID | 自增编号 |
| inpat_id | integer | 住院就诊号 | 关联visit.inpat_record |
| patient_name | varchar | 患者姓名 | 字段名直译 |
| ... | ... | ... | ... |
```

## 7. 与 Reasonix 的协作方式

```
你连接上厂商数据库
       ↓
告诉我：库名 + PostgreSQL 版本 + 已有哪些了解
       ↓
我写探查SQL → 你执行 → 结果发给我
       ↓
我分析表结构、找关联键、推断字段含义
       ↓
输出结构文档（保存到 docs/ 目录）
       ↓
开始写指标SQL
```

> **提示：** 如果字段命名不符合常见规律，或者数据内容看不懂，把采样数据发给我，我帮你推断。
